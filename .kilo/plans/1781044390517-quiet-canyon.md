## ثالثاً: Embedding System

### 1. Embedding Generator
**ملف جديد:** `app/services/embeddings/embedding_generator.py`

**User Embedding** — من:
- UserProfile (gender → one-hot, occupation → one-hot, age → normalized)
- UserQuestionnaireAnswer (كل إجابة → encoded vector)
- UserInteraction (تفضيلات مستنتجة → weighted preferences)

**Property Embedding** — من:
- SyncedProperty (city, government → one-hot, rent → normalized, property_type)
- SyncedRoom (متوسط rent, capacity, ensuite ratio)
- SyncedAmenity (wifi, AC, washer → binary vector)
- Category اللي اتصنف منها

**التقنية:**
- نستخدم `sentence-transformers` (نموذج صغير زي `all-MiniLM-L6-v2`)
- نحول النص الوصفي للـ property/room → text embedding
- ندمج الـ numerical features مع الـ text embedding في vector واحد
- نخزن في جدول `property_embeddings` و `user_embeddings`

### 2. Embedding Pipeline
**ملف جديد:** `app/services/embeddings/pipeline.py`
- **Step 1:** Build feature vectors from structured data
- **Step 2:** Generate or load text embeddings
- **Step 3:** Concatenate features + embedding → final vector
- **Step 4:** Normalize & store in DB
- **Step 5:** periodic refresh (via background task)

## رابعاً: Matching System الجديد (Separate Service)

### 1. Hybrid Matching Engine
**ملف جديد:** `app/services/matching/hybrid_engine.py`

**Hybrid Score** = weighted combination:
```
total = (questionnaire_weight * questionnaire_sim)
      + (embedding_weight * embedding_cosine_sim)
      + (demographics_weight * demographic_sim)
      + (behavior_weight * behavior_sim)
```

**المكونات:**
- **Questionnaire Similarity** — (الموجود حالياً بنفس الوزن 0.35)
- **Embedding Cosine Similarity** — جديد (وزن 0.30)
  - user.euc(user_embedding, roommate.user_embedding)
- **Demographic Similarity** — (gender, occupation, age — موجود حالياً، بنقوّيه)
- **Behavior Similarity** — جديد (وزن 0.15)
  - من UserInteraction: لو الاتنين مهتمين بنفس الـ city/area → score أعلى

### 2. Per-Property Aggregation
**ملف جديد:** `app/services/matching/property_aggregator.py`

لما المستخدم يدور على شقق، نجيب كل الغرف في الشقة، ونحسب matching score بين المستخدم وكل شخص في كل غرفة، ونجمع:
```
property_match_score = avg(
    best_room_score for each room in property
)
```

### 3. API Endpoint
- `POST /match/compute/{user_id}` — يحسب matching لكل الشقق
- `GET /match/property/{property_id}?user_id=...` — matching score لشقة معينة

## سادساً: الـ Data Flow الكامل

```
User Sync (MSSQL → PostgreSQL)
        ↓
Questionnaire Answers
        ↓
Interactions Logged
        ↓
┌─────────────────────────────────────────────┐
│            Background Pipeline                │
│  1. Classify all properties (category)       │
│  2. Generate embeddings (users + properties)  │
│  3. (Future) Train ML model on interactions   │
└─────────────────────────────────────────────┘
        ↓
User Requests Recommendations
        ↓
┌─────────────────────────────────────────────┐
│  GET /recommend/properties/{user_id}         │
│  → PropertyRecommender.recommend()           │
│    - Prefilter (city, budget, category)      │
│    - Score (budget/amenity/location/... )    │
│    - PLUS embedding_score                    │
│    - Rank & Diversity                        │
│  → PropertyAggregator (matching per property)│
│    - لكل شقة، شوف الغرف والناس فيها          │
│    - احسب compatibility score مع كل شخص      │
│    - Aggregate → property_match_score        │
│  → Return: [{property, rec_score, match%}]   │
└─────────────────────────────────────────────┘
```
